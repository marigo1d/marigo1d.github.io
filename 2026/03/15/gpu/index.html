CUDA model

CUDA/编程模型里的内存抽象


kernel <-> grid

一个 Grid通常包含多个 Block（线程块）。

一个 Block（线程块）通常包含多个 Warp（线程束）。


举个具体的例子（以 GA102 为例）：
假设你写了一个 Kernel，每个 Block 需要使用 30KB 的 Shared Memory，包含 256 个线程（即 8 个 Warp）。

  • GigaThread Engine 把 Block 0 发给 SM 1。SM 1 划出 30KB 的 Shared Memory 给 Block 0。
  • SM 1 还有空间，于是把 Block 1 也发给 SM 1。SM 1 又划出 30KB 给 Block 1。
  • 同理，Block 2 也去了 SM 1，再占 30KB。
  • 此时 SM 1 用了 90KB Shared Memory，剩下的不够给下一个 Block 用了。于是 Block 3 会被派发给 SM 2。

注意： Block 0、Block 1、Block 2 都在同一个 SM 1 上运行，它们各自拥有独立的 30KB Shared Memory（互相隔离，不能互访）。这就是为什么 Shared Memory 叫 Block-Level Memory，它的作用域被硬件死死限制在分配给该 Block 的那部分 SRAM 中。
