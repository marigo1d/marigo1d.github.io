Pytorch

维度

张量是有多个“维度”的，每个维度对应一个“方向”：

  • 维度 0： 最外层（最左边的括号）—— 行的个数 → 竖着方向
  • 维度 1： 每一行里的列数 → 横着方向
  • 维度 2： 通常是更深一层，比如 RGB 通道、高度、时间帧等

维度的大小 -> 从括号最外层向内增加

在 PyTorch 中，每多一层括号，维度 +1。

每一层参数（如权重 w, 偏置 b）的梯度会保存在对应的Tensor张量对象中的 .grad 属性里。

next_functions 是一个 tuple，里面的元素代表了 MulBackward0 的 输入函数（上游节点）

PyTorch 中每个计算节点的 grad_fn 是计算图的一部分，它通过 .next_functions 连接上游节点，并在 grad_fn.next_functions._saved_results 中保存前向传播需要的中间变量，反向传播时依此自动计算梯度。

在 PyTorch 中，计算图的清理（Graph Cleanup） 指的是在反向传播完成后，系统自动释放计算图占用的内存，以避免不必要的资源占用。这包括：

  1. 删除中间变量的梯度计算函数（grad_fn）
    • 在反向传播完成后，PyTorch 会自动销毁计算图（除非显式指定 retain_graph=True）。
    • 这意味着所有中间变量（如 z2、a）的 grad_fn 会被置为 None，从而断开计算图的链接。
  2. 释放梯度缓冲区（.grad）
    • 叶子节点（如 w、x、a、b）的梯度（.grad）仍然会保留，除非手动调用 optimizer.zero_grad() 或 tensor.grad = None。
  3. 计算图的内存回收
    • PyTorch 的自动微分引擎（Autograd）会释放计算图占用的内存，避免内存泄漏。

Tensor计算

高维张量的矩阵乘法（如 torch.matmul）

规则：从右向左对齐维度，按矩阵乘法规则扩展。

(1) 核心思想

  • 最后两个维度（行, 列）必须满足矩阵乘法规则：
    (..., m, n) * (..., n, p) → (..., m, p)
    （消去中间的 n）。
  • 其他维度（前面的 ...）必须相同或可广播。

(2) 示例

  1. 批量矩阵乘法：
    (2, 3, 4) * (2, 4, 5) → 结果 (2, 3, 5)
    • 前导维度 2 相同，
    • 最后两维 (3,4) * (4,5) → (3,5)。
  2. 广播矩阵乘法：
    (3, 1, 4) * (1, 4, 5) → 广播为 (3, 1, 4) * (3, 4, 5) → 结果 (3, 1, 5)
    • 前导维度 3 和 1 广播为 3，
    • 最后两维 (1,4) * (4,5) → (1,5)。
